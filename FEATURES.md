# Функциональность CRM-системы ломбарда

## Реализованные функции согласно отчету

### 1. Управление клиентами ✅
- Создание базы данных клиентов
- Добавление новых клиентов
- Редактирование данных клиентов
- Удаление клиентов
- Просмотр списка всех клиентов

### 2. Регистрация и контроль товаров, принятых в залог ✅
- Оформление займов с указанием:
  - Клиента
  - Размера займа
  - Срока займа
  - Состояния товара (для расчета процента)
  - Информации о товаре (наименование, категория, физическое состояние)
  - Исполнителя (сотрудника)
- Автоматический расчет процента по займу на основе:
  - Состояния товара (5%, 7.5%, 10%, 12.5%, 15%)
  - Срока займа (5, 6.25, 7.5, 8.75, 10 месяцев)
- Отслеживание статуса займов:
  - Активен
  - Выплачен
  - Просрочен
- Просмотр детальной информации о займе
- Отметка займа как выплаченного

### 3. Продажа невостребованного имущества ✅
- Добавление товаров в список невостребованных (из просроченных займов)
- Просмотр списка невостребованных товаров
- Оформление продаж невостребованных товаров
- Указание продавца (менеджера по продажам)
- История всех продаж

### 4. Анализ данных и отчётность по состоянию залогов ✅
- Квартальные отчеты с информацией:
  - Общее количество займов
  - Общая сумма займов
  - Количество выплаченных займов
  - Количество просроченных займов
  - Количество продаж
  - Сумма продаж
- Статистика по статусам займов (визуализация)
- Фильтрация отчетов по кварталам и годам

### 5. Формирование квартальных ведомостей ✅
- Формирование отчетов за любой квартал
- Выбор года и квартала
- Детальная статистика по выбранному периоду

### 6. Управление сотрудниками ✅
- Добавление сотрудников с указанием:
  - ФИО
  - Должности (Оценщик-товаровед, Менеджер-товаровед, Менеджер по продажам)
  - Даты приема
  - Телефона
- Увольнение сотрудников
- Просмотр списка всех сотрудников
- Фильтрация активных сотрудников

## Дополнительные функции

### Панель управления
- Общая статистика на главной странице:
  - Количество клиентов
  - Количество активных займов
  - Количество просроченных займов
  - Количество продаж
  - Количество сотрудников

### Современный интерфейс
- Адаптивный дизайн
- Интуитивная навигация
- Визуализация данных (графики)
- Удобные формы ввода
- Уведомления об успешных операциях и ошибках

### Безопасность данных
- Валидация входных данных
- Проверка ограничений базы данных
- Обработка ошибок

## Структура базы данных

Все таблицы созданы согласно отчету:
1. **Клиент** - ID_Клиента, ФИО, Телефон
2. **Сотрудник** - ID_Сотрудника, ФИО_Сотрудника, Должность, Дата_Приёма, Дата_Увольнения, Телефон_Сотрудника
3. **Процент_по_займу** - Индекс_процента, Состояние_товара, Срок_займа, Процент
4. **Займ** - Код_займа, Дата_займа, Клиент, Размер_займа, Процент_по_займу, Срок_займа, Состояние_товара, Артикул_товара, Наименование_товара, Категория_товара, Физическое_состояние, Исполнитель
5. **Невостребованный_товар** - Артикул, Займ, Оценочная_стоимость
6. **Продажа** - Код_продажи, Дата_продажи, Артикул_проданного_товара, Продавец
